<!--kg-card-begin: markdown--><h1>Introduction</h1>  <p>You are creating a PHP website on Windows Azure and need to install additional PHP extensions not provided by default (like MongoDB or IonCube), you’ve come to the right post. It is quite easy.</p>  <h1>Downloading and uploading the extensions</h1>  <p>I’m going to demo this with 2 extensions; MongoDB and IonCube. The first is a regular PHP extension, and the second one is a Zend extension.</p>  <p>It is important that you download the <strong>32-bit version</strong> of the DLLs (unless you are using a Standard website instance and specified that it runs on 64-bit), and also download the <strong>Non Thread Safe versions</strong> (if provided).</p>  <ol>   <li>Once you have the DLLs in hand, create a folder called <strong>bin</strong> under <strong>/site/wwwroot</strong> using your favorite FTP client (or Web Matrix)</li>    <li>Upload the DLLs to the bin folder.</li> </ol>  <h1>Configuring the website on the portal to load the extensions</h1>  <ol>   <li>Go to the Windows Azure portal</li>    <li>Select your website and go to the Configure tab, scroll down till you reach the App Settings section</li>    <li>For PHP extensions, create a <strong>PHP_EXTENSIONS</strong> key, for Zend extensions, create a <strong>PHP_ZENDEXTENSIONS</strong> key</li>    <li>For values, type in the locations of the DLLs as shown in the picture     <br /><a href="__GHOST_URL__/content/images/msdn/0045.image_5F00_437E555A.png"><img title="image" style="display: inline; background-image: none;" border="0" alt="image" src="__GHOST_URL__/content/images/msdn/4214.image_5F00_thumb_5F00_27DC5A15.png" width="644" height="191" /></a></li>    <li>Save the configuration, and load up your phpinfo test page, behold:     <br /><a href="__GHOST_URL__/content/images/msdn/4645.image_5F00_33087510.png"><img title="image" style="display: inline; background-image: none;" border="0" alt="image" src="__GHOST_URL__/content/images/msdn/6747.image_5F00_thumb_5F00_7B55BADF.png" width="644" height="96" /></a>      <br /><a href="__GHOST_URL__/content/images/msdn/1016.image_5F00_0681D5DB.png"><img title="image" style="display: inline; background-image: none;" border="0" alt="image" src="__GHOST_URL__/content/images/msdn/2364.image_5F00_thumb_5F00_11ADF0D6.png" width="644" height="373" /></a></li>    <li>If you want to load multiple extensions, just comma separate them in the App Settings <strong>but make sure not to leave any spaces between the comma separated paths</strong>, otherwise the platform won’t be able to find the file and load it.</li> </ol><blockquote class='note original-post'><div><p><strong>Note: </strong>This post originally appeared on my MSDN blog at 		<a href='http://blogs.msdn.com/b/africaapps/archive/2013/12/23/installing-php-and-zend-extensions-on-windows-azure-websites.aspx'>http://blogs.msdn.com/b/africaapps/archive/2013/12/23/installing-php-and-zend-extensions-on-windows-azure-websites.aspx</a></p></div></blockquote><!--kg-card-end: markdown-->
