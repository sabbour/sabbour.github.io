<!--kg-card-begin: markdown--><h1>Introduction</h1>  <p>In the previous post, we learned how to <a href="__GHOST_URL__/how-to-customize-a-virtual-machine-on-windows-azure-then-capture-it-as-reusable-image">customize a virtual machine and capture it as a reusable image</a>. In this post we are going to be building on top of that and see how we can quickly create a server farm of multiple servers that are load balanced.</p>  <h1>Background</h1>  <p>What is load balancing? When you scale your app to be served from multiple machines, you need something called a load balancer, which is a piece of hardware (or software) that receives incoming requests to your application, and selects which machine will serve such request.</p>  <p>&amp;#160;</p>  <p><a href="__GHOST_URL__/content/images/msdn/8865.lcpod5tw_5F00_33D6DE41.jpg"><img title="lcpod5tw" style="display: inline; background-image: none;" border="0" alt="lcpod5tw" src="__GHOST_URL__/content/images/msdn/5518.lcpod5tw_5F00_thumb_5F00_2A2E700B.jpg" width="827" height="471" /></a></p>  <h1>Let’s start</h1>  <p>I’m assuming you followed the previous post and created a customized image. Open up the management portal and create a new virtual machine from the gallery.</p>  <p><a href="__GHOST_URL__/content/images/msdn/1460.image_5F00_4EDF8782.png"><img title="image" style="display: inline; background-image: none;" border="0" alt="image" src="__GHOST_URL__/content/images/msdn/2311.image_5F00_thumb_5F00_4C5655C4.png" width="644" height="427" /></a></p>  <p>When the gallery opens, select “My Images” then select the image we customized earlier.</p>  <p><a href="__GHOST_URL__/content/images/msdn/4338.image_5F00_2A1E4A3E.png"><img title="image" style="display: inline; background-image: none;" border="0" alt="image" src="__GHOST_URL__/content/images/msdn/4774.image_5F00_thumb_5F00_1C6B8E36.png" width="644" height="428" /></a></p>    <p>Enter the details of the first server in the farm and </p>  <p><a href="__GHOST_URL__/content/images/msdn/0028.image_5F00_334A5FB2.png"><img title="image" style="display: inline; background-image: none;" border="0" alt="image" src="__GHOST_URL__/content/images/msdn/3034.image_5F00_thumb_5F00_312D60E9.png" width="644" height="466" /></a></p>  <p>Create it as a “Stand-alone Virtual Machine”</p>  <p><a href="__GHOST_URL__/content/images/msdn/4555.image_5F00_0F618858.png"><img title="image" style="display: inline; background-image: none;" border="0" alt="image" src="__GHOST_URL__/content/images/msdn/7242.image_5F00_thumb_5F00_3B31DC47.png" width="644" height="465" /></a></p>  <p>While you’re at it, create an Availability Set for the machine too. The availability set feature allows VMs deployed in the same set to be created on different server racks in the data center. Also being in an availability set means that whenever your VMs are taken down for maintenance or upgrades, they will never be taken down simultaneously, and your application will remain available.</p>  <p><a href="__GHOST_URL__/content/images/msdn/7026.image_5F00_3D8B5E45.png"><img title="image" style="display: inline; background-image: none;" border="0" alt="image" src="__GHOST_URL__/content/images/msdn/0284.image_5F00_thumb_5F00_22728F37.png" width="644" height="466" /></a></p>  <p>After the machine is provisioned, let’s create the second one, using the same procedure, but we’ll only change one step. Instead of creating it as a stand alone virtual machine, we will “Connect to an existing Virtual Machine” and pick the machine we created earlier. This will create the 2 virtual machines in the same network.</p>  <p><a href="__GHOST_URL__/content/images/msdn/5344.image_5F00_39BD93A8.png"><img title="image" style="display: inline; background-image: none;" border="0" alt="image" src="__GHOST_URL__/content/images/msdn/0511.image_5F00_thumb_5F00_3396470D.png" width="644" height="464" /></a></p>  <p>And choose the same Availability Set</p>  <p><a href="__GHOST_URL__/content/images/msdn/7220.image_5F00_5CBDDF4B.png"><img title="image" style="display: inline; background-image: none;" border="0" alt="image" src="__GHOST_URL__/content/images/msdn/3566.image_5F00_thumb_5F00_13B7BD85.png" width="644" height="468" /></a></p>    <h1>Load balancing</h1>  <p>Remember in part one, we configured this virtual machine to be an IIS webserver. We will now need to open up the required ports (port 80) in order to make the machines accept HTTP traffic. In doing so, we will also configure load balancing in a moment.</p>  <p>Go to the “Endpoints” tab in the first machine, add a new endpoint listening on public and private ports 80 and save.</p>  <p><a href="__GHOST_URL__/content/images/msdn/8787.image_5F00_6DE19721.png"><img title="image" style="display: inline; background-image: none;" border="0" alt="image" src="__GHOST_URL__/content/images/msdn/6683.image_5F00_thumb_5F00_2547A850.png" width="644" height="464" /></a></p>  <p>After your changes are applied, go to the “Endpoints” tab in the second machine and add a new endpoint, but this time, select the “Load-Balance Traffic on an Existing Endpoint” option, and select the endpoint we created earlier. </p>  <p><a href="__GHOST_URL__/content/images/msdn/3718.image_5F00_6A7FFF79.png"><img title="image" style="display: inline; background-image: none;" border="0" alt="image" src="__GHOST_URL__/content/images/msdn/6404.image_5F00_thumb_5F00_6CD98177.png" width="644" height="465" /></a></p>  <p>That’s it! You’ve created a load balanced web-farm of 2 machines. Now I want to be clear, if your web application doesn’t need to be running in a virtual machine, and can run in a Windows Azure Cloud Service or a Windows Azure Website, your scaling options become much simpler. You simply drag a slider and everything else just falls into place.</p>  <p>This can be the topic of another post though <img class="wlEmoticon wlEmoticon-smile" style="style" alt="Smile" src="__GHOST_URL__/content/images/msdn/6813.wlEmoticon_2D00_smile_5F00_5FFF2B59.png" />.</p><blockquote class='note original-post'><div><p><strong>Note: </strong>This post originally appeared on my MSDN blog at 		<a href='http://blogs.msdn.com/b/africaapps/archive/2013/05/30/creating-a-load-balanced-highly-available-iis-web-farm-on-windows-azure.aspx'>http://blogs.msdn.com/b/africaapps/archive/2013/05/30/creating-a-load-balanced-highly-available-iis-web-farm-on-windows-azure.aspx</a></p></div></blockquote><!--kg-card-end: markdown-->
