<!--kg-card-begin: markdown--><h1>Introduction</h1>  <p>In the <a href="__GHOST_URL__/creating-a-wimp-stack-windows-server-iis-mysql-php-on-windows-azure" target="_blank">previous post</a>, I’ve shown you how to create a WIMP stack (Windows, IIS, MySQL, PHP) on a Windows Azure virtual machine. In this post, we’ll be creating a more traditional LAMP server, but we won’t build things from scratch.</p>  <p>I’ll be using the excellent image provided by <a href="http://bitnami.com/stack/lamp" target="_blank">BitNami</a> which provides a complete PHP, MySQL and Apache environment and comes bundled with phpMyAdmin, SQLite, Varnish, ImageMagick, ModSecurity, XDebug, Xcache, OAuth, Memcache, FastCGI, APC, GD, OpenSSL, CURL, openLDAP, PEAR, PECL and other components and the following frameworks: <a href="http://wiki.bitnami.com/Components/PHP_Frameworks/Zend_Framework">Zend Framework</a>, <a href="http://wiki.bitnami.com/Components/PHP_Frameworks/Symfony">Symfony</a>, <a href="http://wiki.bitnami.com/Components/PHP_Frameworks/CodeIgniter">CodeIgniter</a>, <a href="http://wiki.bitnami.com/Components/PHP_Frameworks/CakePHP">CakePHP</a>, <a href="http://wiki.bitnami.com/Components/PHP_Frameworks/Smarty">Smarty</a> and <a href="http://wiki.bitnami.com/Components/PHP_Frameworks/Laravel">Laravel</a>.</p>  <h1>Getting the image</h1>  <p>Login to the portal and head to Virtual Machines, click on Images then Browse VM Depot</p>  <p><a href="__GHOST_URL__/content/images/msdn/5126.image_5F00_7207E0F6.png"><img title="image" style="display: inline; background-image: none;" border="0" alt="image" src="__GHOST_URL__/content/images/msdn/5265.image_5F00_thumb_5F00_659C4E7E.png" width="644" height="412" /></a></p>  <p>Find the LAMP stack image </p>  <p><a href="__GHOST_URL__/content/images/msdn/6622.image_5F00_24FC92C0.png"><img title="image" style="display: inline; background-image: none;" border="0" alt="image" src="__GHOST_URL__/content/images/msdn/7178.image_5F00_thumb_5F00_3D228DCC.png" width="626" height="484" /></a></p>  <p>Select the datacenter where you will be creating this VM in, then hit the check mark to begin </p>  <p><a href="__GHOST_URL__/content/images/msdn/7065.image_5F00_0335DB91.png"><img title="image" style="display: inline; background-image: none;" border="0" alt="image" src="__GHOST_URL__/content/images/msdn/8015.image_5F00_thumb_5F00_049C8F14.png" width="624" height="484" /></a></p>  <p>Go grab a coffee and come back, it will take about 20 minutes to download to your storage account. It is a 30GB image <img class="wlEmoticon wlEmoticon-smile" style="style" alt="Smile" src="__GHOST_URL__/content/images/msdn/1513.wlEmoticon_2D00_smile_5F00_066F758C.png" /></p>  <p>Once the image is downloaded, you will have to Register it in order to appear on the list of images you can create a Virtual Machine from.</p>  <p><img src="__GHOST_URL__/content/images/msdn/3301.3.1-register.PNG" width="640" height="425" /></p>    <h1>Creating the Virtual Machine</h1>  <p>Now that the image is registered, you can now create Virtual Machines out of it. Click on New and choose to create from gallery, click on My Images, then choose your image to proceed</p>  <p><a href="__GHOST_URL__/content/images/msdn/1488.image_5F00_4875E4CD.png"><img title="image" style="display: inline; background-image: none;" border="0" alt="image" src="__GHOST_URL__/content/images/msdn/5582.image_5F00_thumb_5F00_0287425E.png" width="644" height="413" /></a></p>  <p>Go through the wizard, and open up the HTTP port 80 endpoint while you’re at it, in order to be able to access whatever we are going to host in there</p>  <p><a href="__GHOST_URL__/content/images/msdn/5556.image_5F00_21CC79E2.png"><img title="image" style="display: inline; background-image: none;" border="0" alt="image" src="__GHOST_URL__/content/images/msdn/6428.image_5F00_thumb_5F00_559700E4.png" width="644" height="412" /></a></p>  <p>And you’re done. It should take around 10 minutes to be provisioned. After that, if you open up the DNS name you specified (for example <a href="http://lampstacksabbour.cloudapp.net">http://lampstacksabbour.cloudapp.net</a>), you should be presented with the welcome screen of the BitNami LAMP stack</p>  <p><a href="__GHOST_URL__/content/images/msdn/3187.image_5F00_302F9E27.png"><img title="image" style="display: inline; background-image: none;" border="0" alt="image" src="__GHOST_URL__/content/images/msdn/2577.image_5F00_thumb_5F00_23C40BAF.png" width="644" height="354" /></a></p>  <h1>Accessing the machine</h1>  <h2>SSH access</h2>  <p>Use your favorite SSH client on Windows/Linux/Mac to connect you your machine. I’m using PuTTy to SSH into the VM at lampstackserver.cloudapp.net. Use the same username and password you specified when provisioning the VM in the previous step.</p>  <p><a href="__GHOST_URL__/content/images/msdn/8625.image_5F00_4E32CD7D.png"><img title="image" style="display: inline; background-image: none;" border="0" alt="image" src="__GHOST_URL__/content/images/msdn/0763.image_5F00_thumb_5F00_54E867B1.png" width="644" height="405" /></a></p>  <h2>SFTP access</h2>  <p>To upload your stuff, you should connect via Secure FTP (SFTP). I’m using FileZilla to do that.</p>  <p>Open the Site Manager, specify the hostname, choose SFTP as the protocol and put in your username and password for the machine (or not, depending on whether you want to be asked every time), hit Connect.</p>  <p><a href="__GHOST_URL__/content/images/msdn/2262.image_5F00_1852F9C5.png"><img title="image" style="display: inline; background-image: none;" border="0" alt="image" src="__GHOST_URL__/content/images/msdn/7838.image_5F00_thumb_5F00_0CBFCD37.png" width="644" height="450" /></a></p>  <p>You will see the structure of the folders. It might be a bit confusing at the start but I’ll take you through it later. You can also check out the <a href="http://wiki.bitnami.com/Components" target="_blank">documentation at BitNami</a> and they will help you tremendously.</p>  <p><a href="__GHOST_URL__/content/images/msdn/8715.image_5F00_4500D500.png"><img title="image" style="display: inline; background-image: none;" border="0" alt="image" src="__GHOST_URL__/content/images/msdn/3808.image_5F00_thumb_5F00_64460C84.png" width="559" height="484" /></a></p>  <h2>phpMyAdmin access</h2>  <p>The image comes preinstalled with some frameworks and tools. One of them is phpMyAdmin. Due to security reasons, the phpMyAdmin app is only accessible through the machine itself. If you try to access it, you will get the following message:</p>  <p><a href="__GHOST_URL__/content/images/msdn/5670.image_5F00_18109387.png"><img title="image" style="display: inline; background-image: none;" border="0" alt="image" src="__GHOST_URL__/content/images/msdn/3377.image_5F00_thumb_5F00_27B32F49.png" width="644" height="34" /></a></p>  <p>To solve this, we will need to establish an <a href="http://wiki.bitnami.com/Components/phpMyAdmin_and_phpPgAdmin" target="_blank">SSH tunnel between our computer and the VM. You can follow the documentation</a> on how to easily do that for Linux and Mac. Below I’ll do it for Windows.</p>  <p>Open up PuTTy like before and specify the hostname</p>  <p><a href="__GHOST_URL__/content/images/msdn/2766.image_5F00_5704A4D3.png"><img title="image" style="display: inline; background-image: none;" border="0" alt="image" src="__GHOST_URL__/content/images/msdn/1157.image_5F00_thumb_5F00_61C48CD9.png" width="538" height="484" /></a></p>  <p>Then expand the SSH node, Tunnels then type in the remote port as <strong>8888</strong> and destination as <strong>localhost:80</strong>. Click Add.</p>  <p><a href="__GHOST_URL__/content/images/msdn/3755.image_5F00_3CC95D11.png"><img title="image" style="display: inline; background-image: none;" border="0" alt="image" src="__GHOST_URL__/content/images/msdn/2072.image_5F00_thumb_5F00_6BAE9FA6.png" width="540" height="484" /></a></p>  <p>Go back to the Session node and Save this session, so that you don’t have to redo this every time, then click Open.</p>  <p>Login as usual to the SSH, then you will be able to access phpMyAdmin through <a href="http://127.0.0.1:8888/phpmyadmin">http://127.0.0.1:8888/phpmyadmin</a> </p>  <p>The default username of the server is <strong>root</strong> and the password is <strong>bitnami</strong>.</p>  <h2>Deploying your app</h2>  <p>I won’t go into much details about how the BitNami LAMP stack is structured, <a href="http://wiki.bitnami.com/Components" target="_blank">the docs are excellent</a>, but basically:</p>  <ul>   <li>Your main installation directory is <strong>/opt/bitnami</strong></li>    <li>The script that starts and stops everything is located in <strong>/opt/bitnami/ctlscript.sh</strong></li>    <li>Apache is installed in <strong>/opt/bitnami/apache2</strong></li>    <li>To restart a service, Apache for example, run <strong>sudo /opt/bitnami/ctlscript.sh restart apache</strong></li>    <li>Apache configuration files are located here <strong>/opt/bitnami/apache2/conf</strong></li>    <li>PHP is installed in <strong>/opt/bitnami/php</strong></li>    <li>php.ini is located here <strong>/opt/bitnami/php/etc/php.ini</strong></li> </ul>  <p>There are a couple of <a href="http://wiki.bitnami.com/Components/PHP_Frameworks" target="_blank">PHP frameworks</a> that come preinstalled on this VM:</p>  <ul>   <li><a href="http://wiki.bitnami.com/Components/PHP_Frameworks/CakePHP">CakePHP</a></li>    <li><a href="http://wiki.bitnami.com/Components/PHP_Frameworks/CodeIgniter">CodeIgniter</a></li>    <li><a href="http://wiki.bitnami.com/Components/PHP_Frameworks/Laravel">Laravel</a></li>    <li><a href="http://wiki.bitnami.com/Components/PHP_Frameworks/Smarty">Smarty</a></li>    <li><a href="http://wiki.bitnami.com/Components/PHP_Frameworks/Symfony">Symfony</a></li>    <li><a href="http://wiki.bitnami.com/Components/PHP_Frameworks/Zend_Framework">Zend Framework</a></li> </ul>  <p>The links above will take you to how to setup an app based on those frameworks.</p>  <p>If you are not using any framework and want to <a href="http://wiki.bitnami.com/Applications/BitNami_Custom_PHP_application" target="_blank">deploy a “raw” PHP app</a>, you should put it under the /opt/bitnami/apps folder. Follow the <a href="http://wiki.bitnami.com/Applications/BitNami_Custom_PHP_application" target="_blank">documentation</a> for clear directions.</p>  <h1>Closing</h1>  <p>You might have thought that Windows Azure can’t run the “open source stack”. I hope that with this blog post, I’ve demonstrated that this is indeed not true. We’ve created a fully functional LAMP server out of a preconfigured image from VM Depot. If you prefer to do things from scratch, you can definitely create an empty Ubuntu/OpenSUSE/CentOS/… machine and SSH into it and install whatever you want.</p><blockquote class='note original-post'><div><p><strong>Note: </strong>This post originally appeared on my MSDN blog at 		<a href='http://blogs.msdn.com/b/africaapps/archive/2013/11/07/creating-a-lamp-stack-linux-apache-mysql-php-on-windows-azure.aspx'>http://blogs.msdn.com/b/africaapps/archive/2013/11/07/creating-a-lamp-stack-linux-apache-mysql-php-on-windows-azure.aspx</a></p></div></blockquote><!--kg-card-end: markdown-->
